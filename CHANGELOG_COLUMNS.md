# Обновление: Настраиваемая ширина колонок и сохранение настроек

В этом обновлении была реализована полноценная система управления колонками таблицы транзакций, аналогичная поведению в профессиональных табличных редакторах.

## Реализованные функции

### 1. Интерактивный ресайз колонок
- Добавлены визуальные разделители между столбцами.
- Реализовано плавное изменение ширины с помощью мыши (drag-and-drop).
- **Ограничения**: Добавлен лимит на расширение колонок вправо. Таблица никогда не выходит за пределы экрана и не вызывает появление горизонтального скролла. Последняя колонка (`Change`) выступает в роли "буфера".

### 2. Умное скрытие колонки "Cur. Price"
- При отключении параметра "Show Current Price column" в настройках, ширина скрытой колонки автоматически прибавляется к колонке "Coin / Date".
- Это позволяет столбцам `Holdings`, `Buy Price` и `Profit/Loss` сдвигаться вправо, сохраняя структуру таблицы целостной.
- При включении колонки обратно, все размеры возвращаются к исходным значениям.

### 3. Постоянство настроек (Persistence)
- Все изменения ширины колонок автоматически сохраняются в файл `settings.json` на бэкенде.
- Статус чекбокса "Show Current Price column" теперь также сохраняется в `settings.json`, а не только в LocalStorage.
- При перезапуске приложения все размеры и настройки видимости восстанавливаются в точности так, как их оставил пользователь.

### 4. Оптимизированные параметры по умолчанию
Если файл настроек отсутствует, приложение открывается с выверенными значениями ширины:
- **Logo**: 50px
- **Coin / Date**: 444px
- **Holdings**: 172px
- **Price**: 176px
- **Cur. Price**: 121px
- **Profit/Loss**: 133px
- **Change**: Автоматическая ширина (резиновая).

## Технические изменения

### Бэкенд (Rust/Tauri)
- Обновлена структура `AppSettings` в `settings.rs`.
- Добавлены новые команды в `lib.rs`: `save_column_widths` и `save_show_cur_price`.

### Фронтенд
- **CSS**: Перевод таблицы на `table-layout: fixed` для точного контроля пикселей. Использование `box-sizing: border-box` для предотвращения скачков при захвате границ.
- **JS**: Новая логика в объекте `UI` для обработки событий мыши и интеграция с `AppSettings` для синхронизации с бэкендом.
- **HTML**: Каждой колонке присвоен уникальный ID для точной идентификации в коде.
